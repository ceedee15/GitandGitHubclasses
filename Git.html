Git and GitHub

Git

It is Version control system. If you are doing a project you do it in a version control system. What file created, what file deleted, what are added and deleted and changed in the files are through Git. Not only projects, lot of companies uses Git and Git Hub. Free and open source Fast and scalable – used in small scale as well as big scale projects

We use Git and GitHub for 2 things.

    1. To track the history. For eg, if we are trying to create a website, we are adding features for website one after the other. First we will add the sign up page, then add buttons, then add help form. While adding this help form feature takes time, so you decided to work on this feature then add later and you want the website to be as before. So we have to sit and delete the code line by line manually. It is risky to delete manually. We need a automatic software and we also should know how much lines of codes have written for the website till the buttons got added. We should have that history. So Git helps in tracking that history for the coding of a website.
    2. To collaborate – Think you work as a team in your job, all the developers or team members work on one project. It is important to track the features added in the project or what all changes happened in the project by whom. Also, the changes should not be overridden by others. So Git helps to resolve this problem and in collaborating.

GitHub

Git is a toll that runs in our PC. GitHub is a website. Developers can upload their changes, do changes in the code at Git. If we do any changes in our system, that change can reflect in the GitHub website. We upload the projects in folder form and we call that folder as repository. In short we call it as repos.

Tools Required:

    1. Download Git bash,
    2. Create GitHub Account,
    3. Download VS Code

Commands

    The following commands are used commonly in Git.
    1. git clone (http:) - To clone the repository from GitHub to Git.
    2. git add - This command is used to add the changes made in the file.
    3. git add . - This command is used to add the overall changes in the reposiory.
    4. git commit -m "your commit message" - this command is used when you add the change and to make it visible in the repository.
    5. git push origin (any branch) - To push the changes from the git to GitHub.
    6. git pull - to pull the changes.
    7. git checkout (branch) - To go that particular branch in the repository.
    8. cd (repository) - To go to that particular repository or location or folder.
    9. git merge (branch) - To merge the branches in the repository.
    10. ls - Lists out the files in the repository.
    11. git config --global user.name - To add the username for git.
    12. git config --global user.email - To add the email.
    13. clear - To clear the commands from the terminal.
    14. git status - To show the status of repository. If any changes made in Git and not commited, it will reflect in the status.
    15. git add (file name) - To add changes made in a particular file.
    16. git init - To initialise the folder created.
    17. cd .. - to move back from a repository.

Steps to follow:

    1. Create a new repository in GitHub - GitandGitHubClasses.
    2. Created new file - Git.html
    3. Created 2 branches from main branch - AT and Sophonie
    4. Go to VS code and give git clone (http address)
    5. cd GitandGitHubClasses (go to that repository)
    6. git branch -a (to know what all branches added in the repo)
    7. git checkout AT (to go AT branch)
    8. Add some infos in the created Git.html file
    9. git add . (to add the changes in the repo)
    10. git status (to check the status of the modified file)
    11. git commit -m "added paragraphs" (to commit the changes in the file)
    12. git push origin AT (to make the changes visible in the GitHub)
    13. git checkout main (to go to main branch)
    14. git merge AT (merge AT branch with main branch)
    15. git push origin main (to make the changes in the AT branch appear in main branch)
    16. git checkout sophonie (to go to Sophonie branch)
    17. git pull origin main (to pull the changes from the main branch to sophonie branch)
    18. Add some more paragraph in the HTML file
    19. Git add . (add the changes)
    20. Git commit -m “added git description” ( to commit the changes)
    21. Git push origin Sophonie ( to make the changes appear in the GitHub page)
    22. Git checkout main (go to main branch)
    23. Git merge Sophonie (merge Sophonie branch with main branch)
    24. Git push origin main ( to make all the changes in the Sophonie branch to appear in the main branch)
